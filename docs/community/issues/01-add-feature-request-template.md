# Add a feature request issue template

Labels: `good first issue`, `documentation`, `help wanted`

## Summary

The repo has a bug report template, but not a matching feature request template. Add one so users can suggest new API, deployment, and reader-integration ideas in a more structured way.

## Why this matters

Better issue intake makes it easier to turn community ideas into roadmap items and reduces back-and-forth with reporters.

## Suggested files

- `.github/ISSUE_TEMPLATE/feature_request.md`
- Optionally `.github/ISSUE_TEMPLATE/config.yml`

## Acceptance criteria

- A feature request template exists with sections for problem, proposed solution, alternatives, and deployment context.
- The wording matches the tone of the existing bug report template.
- If `config.yml` is added, it should keep the bug report option visible.

## Nice to have

- Include a short prompt for screenshots or API examples when relevant.
